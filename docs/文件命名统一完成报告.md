## 📋 命名规范

### 核心原则

1. ✅ **中文优先** - 面向中文用户
2. ✅ **清晰描述** - 文件名体现内容
3. ✅ **一致性** - 同类文档命名风格统一
4. ✅ **简洁性** - 避免冗长和复杂缩写

### 保持英文的例外

1. **产品/项目名称**: CodeBuddy, cLLM, Kylin
2. **库/框架名称**: LibTorch, SentencePiece, HuggingFace, Tokenizers
3. **通用技术术语**: HTTP, KV, API, JSON, YAML
4. **代码模块名**: CTokenizer, ModelExecutor, Sampler

### 命名模式

- **模块类**: `{模块名}模块设计.md`
- **指南类**: `{主题}指南.md`
- **报告类**: `{主题}{类型}报告.md`
- **测试类**: `{模块名}测试{类型}.md`
- **设计类**: `{主题}设计.md`
